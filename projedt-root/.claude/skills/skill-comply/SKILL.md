# skill-comply: 自動コンプライアンス測定

コーディングエージェントがスキル、ルール、またはエージェント定義に実際に従っているかを測定します：
1. 任意の.mdファイルから期待される行動シーケンス（仕様）を自動生成
2. プロンプト厳密度を段階的に下げたシナリオを自動生成（支持的→中立→競合的）
3. `claude -p`を実行し、stream-json経由でツールコールトレースをキャプチャ
4. LLMを使用してツールコールを仕様ステップに対して分類（正規表現ではなく）
5. 時間的順序を決定論的にチェック
6. 仕様、プロンプト、タイムラインを含む自己完結型レポートを生成

## 対象ターゲット

- **スキル** (`skills/*/SKILL.md`): search-first、TDDガイドなどのワークフロースキル
- **ルール** (`rules/common/*.md`): testing.md、security.md、git-workflow.mdなどの必須ルール
- **エージェント定義** (`agents/*.md`): 期待通りにエージェントが呼び出されるか（内部ワークフロー検証は未対応）

## 発動タイミング

- ユーザーが`/skill-comply <path>`を実行
- ユーザーが「このルールは実際に遵守されている？」と質問
- 新しいルール/スキルの追加後、エージェントのコンプライアンスを検証
- 品質メンテナンスの一環として定期的に

## 使い方

```bash
# Full run
uv run python -m scripts.run ~/.claude/rules/common/testing.md

# Dry run (no cost, spec + scenarios only)
uv run python -m scripts.run --dry-run ~/.claude/skills/search-first/SKILL.md

# Custom models
uv run python -m scripts.run --gen-model haiku --model sonnet <path>
```

## 重要概念: プロンプト独立性

プロンプトが明示的にサポートしていない場合でも、スキル/ルールが遵守されるかを測定します。

## レポート内容

レポートは自己完結型で、以下を含みます：
1. 期待される行動シーケンス（自動生成された仕様）
2. シナリオプロンプト（各厳密度レベルで何が要求されたか）
3. シナリオごとのコンプライアンススコア
4. LLM分類ラベル付きツールコールタイムライン

### 上級者向け（オプション）

フックに精通したユーザー向けに、コンプライアンスが低いステップに対するフックプロモーション推奨もレポートに含まれます。これは情報提供目的であり、主な価値はコンプライアンスの可視化そのものです。
