# Blueprint — 構築計画ジェネレーター

一行の目的を、あらゆるコーディングエージェントがコールドスタートで実行できるステップバイステップの構築計画に変換します。

## 使用タイミング

- 大規模な機能を明確な依存順序で複数のPRに分割する場合
- 複数セッションにまたがるリファクタリングやマイグレーションを計画する場合
- サブエージェント間で並列ワークストリームを調整する場合
- セッション間のコンテキスト喪失が手戻りを引き起こすタスク

**使用しない場合：** 単一のPRで完了可能なタスク、3回未満のツール呼び出し、またはユーザーが「just do it」と言った場合。

## 動作の仕組み

Blueprintは5フェーズのパイプラインを実行します：

1. **リサーチ** — 事前チェック（git、gh auth、リモート、デフォルトブランチ）、プロジェクト構造・既存計画・メモリファイルを読み込んでコンテキストを収集。
2. **設計** — 目的を1PR単位のステップ（通常3〜12）に分割。依存関係エッジ、並列/直列の順序付け、モデルティア（最強 vs デフォルト）、ステップごとのロールバック戦略を割り当て。
3. **下書き** — 自己完結型のMarkdown計画ファイルを`plans/`に書き込む。すべてのステップにコンテキストブリーフ、タスクリスト、検証コマンド、終了基準を含む — 新しいエージェントが前のステップを読まずに任意のステップを実行可能。
4. **レビュー** — チェックリストとアンチパターンカタログに対して、最強モデルのサブエージェント（例：Opus）に敵対的レビューを委任。確定前にすべての重大な指摘を修正。
5. **登録** — 計画を保存し、メモリインデックスを更新し、ステップ数と並列性の概要をユーザーに提示。

Blueprintはgit/ghの利用可能性を自動検出します。git + GitHub CLIが利用可能な場合、完全なブランチ/PR/CIワークフロー計画を生成します。利用できない場合は、ダイレクトモード（直接編集、ブランチなし）に切り替えます。

## 使用例

### 基本的な使用法

```
/blueprint myapp "migrate database to PostgreSQL"
```

`plans/myapp-migrate-database-to-postgresql.md`が以下のようなステップで生成されます：
- ステップ1: PostgreSQLドライバーと接続設定の追加
- ステップ2: 各テーブルのマイグレーションスクリプトの作成
- ステップ3: 新しいドライバーを使用するようリポジトリレイヤーを更新
- ステップ4: PostgreSQLに対するインテグレーションテストの追加
- ステップ5: 旧データベースコードと設定の削除

### マルチエージェントプロジェクト

```
/blueprint chatbot "extract LLM providers into a plugin system"
```

可能な場合は並列ステップを含む計画を生成します（例：「Anthropicプラグインの実装」と「OpenAIプラグインの実装」はプラグインインターフェースのステップ完了後に並列実行）。モデルティアの割り当て（インターフェース設計ステップには最強、実装にはデフォルト）、すべてのステップ後に検証される不変条件（例：「既存のテストがすべてパスする」「コアにプロバイダーのインポートがない」）も含みます。

## 主要機能

- **コールドスタート実行** — すべてのステップに自己完結型のコンテキストブリーフが含まれる。事前コンテキスト不要。
- **敵対的レビューゲート** — すべての計画が、完全性・依存関係の正確性・アンチパターン検出をカバーするチェックリストに対して最強モデルのサブエージェントによりレビューされる。
- **ブランチ/PR/CIワークフロー** — すべてのステップに組み込み。git/ghが存在しない場合はダイレクトモードにグレースフルにフォールバック。
- **並列ステップ検出** — 依存関係グラフが共有ファイルや出力依存のないステップを特定。
- **計画変更プロトコル** — ステップの分割、挿入、スキップ、並べ替え、中止を正式なプロトコルと監査証跡で実行可能。
- **ゼロランタイムリスク** — 純粋なMarkdownスキル。リポジトリ全体が`.md`ファイルのみ — フック、シェルスクリプト、実行可能コード、`package.json`、ビルドステップなし。インストールや呼び出し時にClaude Codeのネイティブなスキルローダー以上のものは何も実行されない。

## インストール

このスキルはEverything Claude Codeに同梱されています。ECCがインストールされている場合、別途インストールは不要です。

### 完全なECCインストール

ECCリポジトリのチェックアウトから作業している場合、以下でスキルの存在を確認してください：

```bash
test -f skills/blueprint/SKILL.md
```

後で更新する場合は、更新前にECCの差分を確認してください：

```bash
cd /path/to/everything-claude-code
git fetch origin main
git log --oneline HEAD..origin/main       # review new commits before updating
git checkout <reviewed-full-sha>          # pin to a specific reviewed commit
```

### ベンダード・スタンドアロンインストール

完全なECCインストール外でこのスキルのみをベンダリングする場合、レビュー済みのECCリポジトリからファイルを`~/.claude/skills/blueprint/SKILL.md`にコピーしてください。ベンダードコピーにはgitリモートがないため、`git pull`ではなくレビュー済みのECCコミットからファイルを再コピーして更新してください。

## 要件

- Claude Code（`/blueprint`スラッシュコマンド用）
- Git + GitHub CLI（オプション — 完全なブランチ/PR/CIワークフローを有効化。Blueprintは不在を検出して自動的にダイレクトモードに切り替え）
