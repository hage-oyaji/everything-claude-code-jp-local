# 投資家向け資料

一貫性があり、信頼性が高く、防御しやすい投資家向け資料を構築します。

## アクティベーション条件

- ピッチデッキの作成または改訂
- 投資家メモまたはワンページャーの作成
- 財務モデル、マイルストーン計画、資金使途表の構築
- アクセラレーターまたはインキュベーターの申請質問への回答
- 複数の資金調達ドキュメントを単一の真実のソースに整合させる

## ゴールデンルール

すべての投資家向け資料は相互に整合していなければなりません。

作成前に単一の真実のソースを作成または確認してください:
- トラクションメトリクス
- 価格設定と収益の前提条件
- 調達額と金融商品
- 資金使途
- チーム経歴と役職
- マイルストーンとタイムライン

矛盾する数値が現れた場合は、ドラフト作成前に解決してください。

## コアワークフロー

1. 正規の事実をインベントリ
2. 欠落している前提条件を特定
3. アセットタイプを選択
4. 明示的なロジックでアセットをドラフト
5. すべての数値を真実のソースとクロスチェック

## アセットガイダンス

### ピッチデッキ
推奨フロー:
1. 会社 + ウェッジ
2. 課題
3. ソリューション
4. プロダクト / デモ
5. 市場
6. ビジネスモデル
7. トラクション
8. チーム
9. 競合 / 差別化
10. アスク
11. 資金使途 / マイルストーン
12. 付録

ウェブネイティブなデッキが必要な場合は、このスキルを`frontend-slides`と組み合わせてください。

### ワンページャー / メモ
- 会社が何をしているかを1文のきれいな文で述べる
- なぜ今かを示す
- トラクションと証拠ポイントを早期に含める
- アスクを正確にする
- 主張を検証しやすくする

### 財務モデル
含めるもの:
- 明示的な前提条件
- 有用な場合はベア / ベース / ブルケース
- レイヤーバイレイヤーのクリーンな収益ロジック
- マイルストーンに連動した支出
- 意思決定が前提条件に依存する場合の感度分析

### アクセラレーター申請
- 質問されたことに正確に答える
- トラクション、インサイト、チームの優位性を優先する
- 誇大表現を避ける
- 内部メトリクスをデッキやモデルと整合させる

## 避けるべきレッドフラグ

- 検証不可能な主張
- 前提条件のないあいまいな市場規模
- 一貫性のないチームの役割や役職
- きれいに合計されない収益計算
- 前提条件が脆弱な部分での過度の確信

## クオリティゲート

納品前に:
- すべての数値が現在の真実のソースと一致
- 資金使途と収益レイヤーが正しく合計される
- 前提条件が埋もれていなく、可視的である
- ストーリーが誇大な言葉なしに明確である
- 最終アセットがパートナーミーティングで防御可能である
