# エージェントハーネス構築

エージェントの計画立案、ツール呼び出し、エラーからの回復、完了への収束を改善する際にこのスキルを使用してください。

## コアモデル

エージェントの出力品質は以下の要素によって制約されます：
1. アクションスペースの品質
2. オブザベーションの品質
3. リカバリーの品質
4. コンテキストバジェットの品質

## アクションスペース設計

1. 安定した明示的なツール名を使用する。
2. 入力はスキーマファーストで狭く保つ。
3. 決定論的な出力形式を返す。
4. 分離が不可能な場合を除き、汎用ツールを避ける。

## 粒度ルール

- ハイリスク操作（デプロイ、マイグレーション、権限）にはマイクロツールを使用する。
- 一般的な編集/読み取り/検索ループにはミディアムツールを使用する。
- ラウンドトリップのオーバーヘッドが支配的なコストである場合にのみマクロツールを使用する。

## オブザベーション設計

すべてのツールレスポンスには以下を含める：
- `status`: success|warning|error
- `summary`: 結果の1行要約
- `next_actions`: 実行可能なフォローアップ
- `artifacts`: ファイルパス / ID

## エラーリカバリー契約

すべてのエラーパスに以下を含める：
- 根本原因のヒント
- 安全なリトライ手順
- 明示的な停止条件

## コンテキストバジェット管理

1. システムプロンプトは最小限で不変に保つ。
2. 大きなガイダンスはオンデマンドで読み込むスキルに移動する。
3. 長いドキュメントのインライン化よりファイル参照を優先する。
4. 任意のトークン閾値ではなくフェーズ境界でコンパクト化する。

## アーキテクチャパターンガイダンス

- ReAct: パスが不確定な探索的タスクに最適。
- Function-calling: 構造化された決定論的フローに最適。
- ハイブリッド（推奨）: ReActプランニング + 型付きツール実行。

## ベンチマーク

追跡項目：
- 完了率
- タスクあたりのリトライ回数
- pass@1およびpass@3
- 成功タスクあたりのコスト

## アンチパターン

- セマンティクスが重複するツールが多すぎる。
- リカバリーヒントのない不透明なツール出力。
- 次のステップのないエラーのみの出力。
- 無関係な参照によるコンテキストの過負荷。
