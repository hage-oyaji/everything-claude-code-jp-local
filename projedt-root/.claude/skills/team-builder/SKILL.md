# チームビルダー

エージェントチームをオンデマンドで閲覧・構成するためのインタラクティブメニュー。フラットまたはドメインサブディレクトリのエージェントコレクションに対応。

## 使用タイミング

- 複数のエージェントペルソナ（マークダウンファイル）があり、タスクに使用するものを選択したいとき
- 異なるドメイン（例: セキュリティ + SEO + アーキテクチャ）からアドホックチームを構成したいとき
- 決定前に利用可能なエージェントを閲覧したいとき

## 前提条件

エージェントファイルはペルソナプロンプト（アイデンティティ、ルール、ワークフロー、成果物）を含むマークダウンファイルである必要があります。最初の`# 見出し`がエージェント名として使用され、最初の段落が説明として使用されます。

フラットとサブディレクトリの両方のレイアウトがサポートされています:

**サブディレクトリレイアウト** — ドメインはフォルダ名から推測:

```
agents/
├── engineering/
│   ├── security-engineer.md
│   └── software-architect.md
├── marketing/
│   └── seo-specialist.md
└── sales/
    └── discovery-coach.md
```

**フラットレイアウト** — 共有ファイル名プレフィックスからドメインを推測。プレフィックスは2つ以上のファイルが共有する場合にドメインとしてカウントされます。ユニークなプレフィックスのファイルは「General」に分類されます。注: アルゴリズムは最初の`-`で分割するため、複数語のドメイン（例: `product-management`）はサブディレクトリレイアウトを使用すべきです:

```
agents/
├── engineering-security-engineer.md
├── engineering-software-architect.md
├── marketing-seo-specialist.md
├── marketing-content-strategist.md
├── sales-discovery-coach.md
└── sales-outbound-strategist.md
```

## 設定

エージェントディレクトリは順序でプローブされ、結果がマージされます:

1. `./agents/**/*.md` + `./agents/*.md` — プロジェクトローカルのエージェント（両方の深さ）
2. `~/.claude/agents/**/*.md` + `~/.claude/agents/*.md` — グローバルエージェント（両方の深さ）

すべてのロケーションからの結果がマージされ、エージェント名で重複排除されます。同じ名前のグローバルエージェントよりもプロジェクトローカルのエージェントが優先されます。ユーザーが指定した場合、カスタムパスも使用可能です。

## 仕組み

### ステップ1: 利用可能なエージェントを検出

上記のプローブ順序でエージェントディレクトリをGlob検索。READMEファイルは除外。見つかった各ファイルについて:
- **サブディレクトリレイアウト:** 親フォルダ名からドメインを抽出
- **フラットレイアウト:** すべてのファイル名プレフィックス（最初の`-`の前のテキスト）を収集。プレフィックスが2つ以上のファイル名に現れる場合のみドメインとして認定（例: `engineering-security-engineer.md`と`engineering-software-architect.md`は両方`engineering`で始まる → Engineeringドメイン）。ユニークなプレフィックスのファイル（例: `code-reviewer.md`、`tdd-guide.md`）は「General」にグループ化
- 最初の`# 見出し`からエージェント名を抽出。見出しがない場合、ファイル名から名前を導出（`.md`を削除、ハイフンをスペースに置換、タイトルケース）
- 見出しの後の最初の段落から一行のサマリーを抽出

すべてのロケーションをプローブした後にエージェントファイルが見つからない場合、ユーザーに通知: 「エージェントファイルが見つかりません。チェック済み: [プローブしたパスのリスト]。期待: これらのディレクトリのいずれかにあるマークダウンファイル。」その後停止。

### ステップ2: ドメインメニューを表示

```
利用可能なエージェントドメイン:
1. Engineering — Software Architect, Security Engineer
2. Marketing — SEO Specialist
3. Sales — Discovery Coach, Outbound Strategist

ドメインを選択するか、特定のエージェントを指定してください（例: "1,3" または "security + seo"）:
```

- エージェントゼロのドメイン（空のディレクトリ）をスキップ
- ドメインごとのエージェント数を表示

### ステップ3: 選択を処理

柔軟な入力を受け付け:
- 番号: 「1,3」でEngineeringとSalesのすべてのエージェントを選択
- 名前: 「security + seo」で検出されたエージェントにファジーマッチ
- 「all from engineering」でそのドメインのすべてのエージェントを選択

5つ以上のエージェントが選択された場合、アルファベット順にリストし、ユーザーに絞り込みを依頼: 「N個のエージェントを選択しました（最大5個）。保持するものを選択するか、'first 5'と言って最初の5つをアルファベット順で使用してください。」

選択を確認:
```
選択済み: Security Engineer + SEO Specialist
何に取り組むべきですか？（タスクを説明してください）:
```

### ステップ4: エージェントを並列で起動

1. 選択された各エージェントのマークダウンファイルを読み取り
2. まだ提供されていない場合、タスク説明を求める
3. エージェントツールを使用してすべてのエージェントを並列で起動:
   - `subagent_type: "general-purpose"`
   - `prompt: "{agent file content}\n\nTask: {task description}"`
   - 各エージェントは独立して実行 — エージェント間通信は不要
4. エージェントが失敗した場合（エラー、タイムアウト、空の出力）、失敗をインラインで記録（例: 「Security Engineer: 失敗 — [理由]」）し、成功したエージェントの結果で続行

### ステップ5: 結果を統合

すべての出力を収集し、統一レポートを提示:
- エージェントごとにグループ化された結果
- 以下をハイライトする統合セクション:
  - エージェント間の合意事項
  - 推奨事項間の対立や緊張
  - 推奨される次のステップ

1つのエージェントのみ選択された場合、統合をスキップして出力を直接提示。

## ルール

- **動的検出のみ。** エージェントリストをハードコードしない。ディレクトリ内の新しいファイルがメニューに自動表示。
- **チームあたり最大5エージェント。** 5を超えると収穫逓減と過度のトークン使用を生む。選択時に適用。
- **並列ディスパッチ。** すべてのエージェントは同時に実行 — エージェントツールの並列呼び出しパターンを使用。
- **並列Agent呼び出し、TeamCreateではなく。** このスキルは独立した作業のために並列Agent呼び出しを使用。TeamCreate（マルチエージェント対話のためのClaude Codeツール）はエージェントが議論や応答し合う必要がある場合のみ必要。

```
Claude:
利用可能なエージェントドメイン:
1. Engineering (2) — Software Architect, Security Engineer
2. Marketing (1) — SEO Specialist
3. Sales (4) — Discovery Coach, Outbound Strategist, Proposal Strategist, Sales Engineer
4. Support (1) — Executive Summary

ドメインを選択するか、特定のエージェントを指定してください:

ユーザー: security + seo

Claude:
選択済み: Security Engineer + SEO Specialist
何に取り組むべきですか？

ユーザー: Review my Next.js e-commerce site before launch

[両方のエージェントが並列で起動、各自の専門分野をコードベースに適用]

Claude:
## Security Engineerの所見
- [所見...]

## SEO Specialistの所見
- [所見...]

## 統合
両エージェントの合意事項: [...]
緊張点: Securityはインラインスタイルをブロックするcsを推奨、SEOはインラインスキーママークアップが必要。解決策: [...]
次のステップ: [...]
```
