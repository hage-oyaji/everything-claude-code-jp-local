# デザインシステム — ビジュアルシステムの生成と監査

## 使用するタイミング

- デザインシステムが必要な新規プロジェクトを開始する時
- 既存コードベースのビジュアル一貫性を監査する時
- リデザイン前 — 現状を把握する時
- UIが「何かおかしい」が原因を特定できない時
- スタイリングに関するPRをレビューする時

## 仕組み

### モード1: デザインシステム生成

コードベースを分析し、統一されたデザインシステムを生成します：

```
1. CSS/Tailwind/styled-componentsの既存パターンをスキャン
2. 抽出: カラー、タイポグラフィ、スペーシング、ボーダー半径、シャドウ、ブレークポイント
3. インスピレーションのために競合サイトを3つリサーチ（ブラウザMCP経由）
4. デザイントークンセットを提案（JSON + CSSカスタムプロパティ）
5. 各決定の根拠を含むDESIGN.mdを生成
6. インタラクティブなHTMLプレビューページを作成（自己完結型、依存関係なし）
```

出力: `DESIGN.md` + `design-tokens.json` + `design-preview.html`

### モード2: ビジュアル監査

UIを10の項目でスコアリング（各0-10点）：

```
1. カラーの一貫性 — パレットを使用しているか、ランダムな16進値か？
2. タイポグラフィの階層 — 明確なh1 > h2 > h3 > body > captionか？
3. スペーシングのリズム — 一貫したスケール（4px/8px/16px）か任意か？
4. コンポーネントの一貫性 — 類似要素が類似して見えるか？
5. レスポンシブ動作 — ブレークポイントで流動的か崩れるか？
6. ダークモード — 完全か途中か？
7. アニメーション — 目的があるか過剰か？
8. アクセシビリティ — コントラスト比、フォーカス状態、タッチターゲット
9. 情報密度 — 雑然か整然か？
10. 仕上げ — ホバー状態、トランジション、ローディング状態、空状態
```

各項目にスコア、具体例、正確なファイル:行を含む修正を提供します。

### モード3: AIスロップ検出

汎用的なAI生成デザインパターンを特定します：

```
- あらゆるものへの過剰なグラデーション
- パープルからブルーのデフォルト
- 目的のない「グラスモーフィズム」カード
- 丸める必要のないものへの角丸
- スクロール時の過剰なアニメーション
- ストックグラデーション上の中央揃えテキストの汎用ヒーロー
- 個性のないサンセリフフォントスタック
```

## 使用例

**SaaSアプリ向けの生成:**
```
/design-system generate --style minimal --palette earth-tones
```

**既存UIの監査:**
```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**AIスロップのチェック:**
```
/design-system slop-check
```
