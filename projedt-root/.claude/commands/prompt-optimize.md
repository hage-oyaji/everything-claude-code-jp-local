# /prompt-optimize

以下のプロンプトを分析し、ECCの活用を最大化するように最適化します。

## タスク

ユーザーの入力に対して**prompt-optimizer**スキルを適用してください。6フェーズの分析パイプラインに従います：

0. **プロジェクト検出** — CLAUDE.mdを読み取り、プロジェクトファイル（package.json、go.mod、pyproject.tomlなど）から技術スタックを検出
1. **意図の検出** — タスクタイプを分類（新機能、バグ修正、リファクタリング、調査、テスト、レビュー、ドキュメント、インフラ、デザイン）
2. **スコープ評価** — 複雑さを評価（TRIVIAL / LOW / MEDIUM / HIGH / EPIC）、コードベースサイズをシグナルとして利用
3. **ECCコンポーネントマッチング** — 特定のスキル、コマンド、エージェント、モデルティアにマッピング
4. **不足コンテキストの検出** — ギャップを特定。重要な項目が3つ以上不足している場合は、生成前にユーザーに確認を依頼
5. **ワークフローとモデル** — ライフサイクル上の位置を決定、モデルティアを推奨、HIGH/EPICの場合は複数プロンプトに分割

## 出力要件

- 診断結果、推奨ECCコンポーネント、およびprompt-optimizerスキルの出力フォーマットに従った最適化プロンプトを提示
- **フルバージョン**（詳細版）と**クイックバージョン**（コンパクト版、意図タイプにより変化）の両方を提供
- ユーザーの入力と同じ言語で応答
- 最適化されたプロンプトは完全で、新しいセッションにコピー＆ペーストできる状態にする
- 調整の提案または別途実行リクエストを開始するための明確な次のステップをフッターに記載

## 重要事項

ユーザーのタスクを実行しないでください。分析と最適化されたプロンプトのみを出力してください。
ユーザーが直接実行を依頼した場合は、`/prompt-optimize`はアドバイザリー出力のみを生成するものであることを説明し、通常のタスクリクエストを開始するように案内してください。

注意：`blueprint`は**スキル**であり、スラッシュコマンドではありません。`/...`コマンドとして提示するのではなく、「blueprintスキルを使用してください」と記述してください。
