# モデルルートコマンド

タスクの複雑さと予算に基づいて、最適なモデルティアを推奨します。

## 使い方

`/model-route [task-description] [--budget low|med|high]`

## ルーティングヒューリスティック

- `haiku`: 決定論的で、リスクの低い機械的変更
- `sonnet`: 実装とリファクタリングのデフォルト
- `opus`: アーキテクチャ、深いレビュー、曖昧な要件

## 必須出力

- 推奨モデル
- 信頼度レベル
- このモデルが適している理由
- 最初の試行が失敗した場合のフォールバックモデル

## 引数

$ARGUMENTS:
- `[task-description]` オプション（フリーテキスト）
- `--budget low|med|high` オプション
