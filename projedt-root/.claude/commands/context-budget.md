# コンテキストバジェットオプティマイザー

Claude Codeセットアップのコンテキストウィンドウ消費量を分析し、トークンオーバーヘッドを削減するための実用的な推奨事項を提示します。

## 使い方

```
/context-budget [--verbose]
```

- デフォルト: 主要な推奨事項を含むサマリー
- `--verbose`: コンポーネントごとの完全な内訳

$ARGUMENTS

## 実行内容

以下の入力で **context-budget** スキル（`skills/context-budget/SKILL.md`）を実行する:

1. `$ARGUMENTS` に `--verbose` フラグがある場合はそれを渡す
2. ユーザーが別途指定しない限り、200Kコンテキストウィンドウ（Claude Sonnetのデフォルト）を想定する
3. スキルの4つのフェーズに従う: インベントリ → 分類 → 問題検出 → レポート
4. フォーマットされたコンテキストバジェットレポートをユーザーに出力する

スキルがすべてのスキャンロジック、トークン推定、問題検出、レポートフォーマットを処理します。
