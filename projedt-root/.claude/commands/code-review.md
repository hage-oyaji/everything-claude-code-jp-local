# コードレビュー

コミットされていない変更に対する包括的なセキュリティと品質のレビュー:

1. 変更ファイルを取得する: git diff --name-only HEAD

2. 各変更ファイルについて、以下を確認する:

**セキュリティの問題 (CRITICAL):**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクションの脆弱性
- XSSの脆弱性
- 入力バリデーションの欠如
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質 (HIGH):**
- 50行を超える関数
- 800行を超えるファイル
- 4レベルを超えるネスト
- エラーハンドリングの欠如
- console.log文
- TODO/FIXMEコメント
- パブリックAPIのJSDoc欠如

**ベストプラクティス (MEDIUM):**
- ミューテーションパターン（代わりにイミュータブルを使用）
- コード/コメントでの絵文字の使用
- 新しいコードのテスト欠如
- アクセシビリティの問題（a11y）

3. 以下を含むレポートを生成する:
   - 重大度: CRITICAL、HIGH、MEDIUM、LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 修正の提案

4. CRITICALまたはHIGHの問題が見つかった場合、コミットをブロックする

セキュリティの脆弱性があるコードを承認してはならない！
