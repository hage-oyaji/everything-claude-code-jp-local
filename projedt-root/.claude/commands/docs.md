# /docs

## 目的

ライブラリ、フレームワーク、またはAPIの最新ドキュメントを検索し、関連するコードスニペットを含む要約された回答を返します。Context7 MCP（resolve-library-idおよびquery-docs）を使用するため、トレーニングデータではなく最新のドキュメントに基づいた回答を提供します。

## 使い方

```
/docs [ライブラリ名] [質問]
```

複数の単語からなる引数はクォートで囲んで、単一のトークンとして解析されるようにしてください。例: `/docs "Next.js" "How do I configure middleware?"`

ライブラリまたは質問が省略された場合、ユーザーに以下を確認する:
1. ライブラリまたはプロダクト名（例: Next.js、Prisma、Supabase）。
2. 具体的な質問またはタスク（例: "How do I set up middleware?"、"Auth methods"）。

## ワークフロー

1. **ライブラリIDの解決** — Context7ツールの `resolve-library-id` をライブラリ名とユーザーの質問で呼び出し、Context7互換のライブラリID（例: `/vercel/next.js`）を取得する。
2. **ドキュメントの検索** — そのライブラリIDとユーザーの質問で `query-docs` を呼び出す。
3. **要約** — 簡潔な回答を返し、取得したドキュメントから関連するコード例を含める。ライブラリ（関連する場合はバージョンも）を記載する。

## 出力

ユーザーは最新のドキュメントに裏付けられた短く正確な回答と、役立つコードスニペットを受け取ります。Context7が利用できない場合はその旨を伝え、ドキュメントが古い可能性がある注記を付けてトレーニングデータから回答します。
