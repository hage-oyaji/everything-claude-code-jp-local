# テスト要件

## 最低テストカバレッジ：80%

テストの種類（すべて必須）：
1. **ユニットテスト** - 個々の関数、ユーティリティ、コンポーネント
2. **インテグレーションテスト** - APIエンドポイント、データベース操作
3. **E2Eテスト** - クリティカルなユーザーフロー（言語ごとにフレームワークを選択）

## テスト駆動開発

必須ワークフロー：
1. まずテストを書く（RED）
2. テストを実行 - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行 - パスするはず
5. リファクタリング（IMPROVE）
6. カバレッジを確認（80%以上）

## テスト失敗のトラブルシューティング

1. **tdd-guide**エージェントを使用
2. テストの分離を確認
3. モックが正しいことを確認
4. 実装を修正する（テストが間違っている場合を除き、テストではなく実装を修正）

## エージェントサポート

- **tdd-guide** - 新機能に対してプロアクティブに使用、テストファーストを強制
